# 从 Express 到 Hexo - 我的博客迁移之路

今天是我的个人博客正式开通的日子。我想记录一下这次博客系统的选型和搭建过程。

---

## 🤔 为什么搭建博客

作为一名有 20 多年经验的软件工程师，我一直想有一个自己的技术分享平台。 reasons:

1. **记录技术成长** - 把工作中的经验和踩过的坑记录下来
2. **知识沉淀** - 系统整理自己的技术体系
3. **分享交流** - 和其他开发者交流学习
4. **个人品牌** - 建立自己的技术影响力

---

## 🎯 技术选型

在搭建博客之前，我面临两个选择：

### 方案 A：动态博客（Express + 数据库）

**技术栈**：
- Node.js + Express
- PostgreSQL 数据库
- 后台管理系统

**优点**：
- 可以在线写文章
- 有完整的后台管理
- 支持用户系统

**缺点**：
- 需要服务器
- 需要维护数据库
- 有成本（约¥50-100/月）

### 方案 B：静态博客（Hexo + GitHub Pages）

**技术栈**：
- Hexo 静态网站生成器
- GitHub Pages 托管
- Markdown 写作

**优点**：
- 完全免费
- 无需维护服务器
- 速度快，CDN 加速

**缺点**：
- 需要本地写文章
- 没有后台管理
- 功能相对简单

---

## ✅ 最终选择

经过考虑，我选择了 **方案 B（Hexo）**，原因是：

1. **零成本启动** - 初期不想投入太多
2. **专注内容** - 不想花时间维护服务器
3. **简单够用** - 个人写作不需要复杂功能
4. **可升级** - 未来需要时可以迁移到动态博客

---

## 🚀 搭建过程

### 1. 安装 Hexo

```bash
npm install -g hexo-cli
```

### 2. 初始化博客

```bash
hexo init myblog
cd myblog
npm install
```

### 3. 配置 GitHub 部署

修改 `_config.yml`：

```yaml
deploy:
  type: git
  repo: git@github.com:codefan/codefan.github.io.git
  branch: master
```

### 4. 创建第一篇文章

```bash
hexo new post "Hello World"
```

### 5. 部署上线

```bash
hexo clean && hexo g -d
```

就这么简单！几分钟后，博客就上线了。

## 📝 写作体验

使用 Hexo 写作是一种很棒的体验：

- **Markdown 格式** - 专注内容，不用管样式

- **版本控制** - 所有文章都有 Git 历史记录
- **快速部署** - 一条命令就上线
